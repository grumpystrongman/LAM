# OpenLAMb Commercial Features

OpenLAMb is no longer just a workflow runner.
Its commercial surface now spans four product layers.

## 1. Operator Runtime

- natural-language instruction routing
- UIA-first desktop execution
- Playwright-first browser execution
- artifact generation and secure local artifact routing
- clipboard capture and reusable action planning
- saved automations and scheduler support

## 2. Mission and Work-Product Layer

- mission contracts and deliverable modes
- evidence mapping and source quality scoring
- work-product generation for trackers, briefs, reports, dashboards, and code workspaces
- artifact-specific critics and revision runtime
- truthful completion status handling

## 3. Teach Mode

- workflow recording
- learned recipe families
- multi-path adaptive replay
- checkpoint-aware fallback and re-teach guidance
- branch health, decay, and family maintenance

## 4. Topic Mastery Learn Mode

- topic-only or seed-video learning
- related source discovery and ranking
- multi-source synthesis and contradiction handling
- reusable learned skill generation
- mastery guide and safe practice runtime
- skill versioning, diff, scheduling, feedback, and refresh

## Governance and Safety

- explicit control grant
- pause/resume for auth or uncertain steps
- risky-action confirmation gates
- local DPAPI-backed password vault
- authenticated control plane API
- durable audit validation

## Commercial Positioning

OpenLAMb is designed for teams and power users who need:
- local-first operation
- real desktop and browser control
- reusable learned skills
- evidence-backed work products
- clearer operational accountability than black-box agent demos
